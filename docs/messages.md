# Карта сообщений бота

Все тексты, которые видит пользователь, по этапам сценария. ID на схемах ведут в таблицы ниже.

Хендлеры зеркальны: каждый текст живёт в `handlers/telegram/…` и `handlers/max/…`, шедулеры (`schedulers/`) шлют в обе платформы. Отличия Max собраны в последнем разделе. Плейсхолдеры показаны как `{…}`. Найти текст в коде — grep по первым словам сообщения. Дословный текст сообщений — моноширинным, обычным шрифтом — пометки (кнопки, условия, варианты).

Актуально на 2026-06-10. При изменении текстов обновлять этот файл.

## Схема 1. Загрузка файла

```mermaid
flowchart TD
    U([Файл от пользователя]) --> ACK["B1 B2 — 📥 Файл получен"]
    ACK -->|больше 20 МБ| DL["B3 — 📥 Скачиваю файл… N%"]
    DL --> CONV
    ACK --> CONV["B4 — 🎬 Извлекаю аудиодорожку… N%"]
    CONV --> SAVE["B5 — ✨ Почти готово…"]
    SAVE --> OK["B6 — 🎧 Аудио подготовлено<br/>кнопки: Распознать / Отменить"]
    OK -->|не хватает баланса| TW["B7 — ⚠️ На балансе не хватает средств"]
    ACK -.->|ошибка скачивания| E1["B8 B9 — ❌ Не удалось загрузить или скачать"]
    ACK -.->|неподдерживаемый тип| E2["B10 — ❌ Тип файла не поддерживается"]
    CONV -.->|нет длительности или слишком длинный| E3["B11 B12 — ❌"]
    CONV -.->|нет звука, битый файл, формат| E4["B13 B14 B15 — ❌"]
    SAVE -.->|ошибка S3| E5["B16 — ❌ Не удалось загрузить файл"]
```

## Схема 2. Распознавание и результат

```mermaid
flowchart TD
    BTN([Кнопка «Распознать»]) --> RUN["C4 — ⏳ Распознаём запись…"]
    BTN -.->|не хватает денег| C2["C2 — ❌ Недостаточно средств"]
    BTN -.->|провайдер не стартовал| C3["C3 — ❌ Не удалось запустить + возврат"]
    CBTN([Кнопка «Отменить»]) --> C1["C1 — ❌ Задача отменена"]
    RUN -->|дольше 15 мин| C5["C5 — плюс извинение за задержку 🙏"]
    C5 --> RUN
    RUN -.->|процесс умер на старте| C6["C6 — ❌ Не удалось запустить + возврат"]
    RUN -.->|дольше 2 ч| C7["C7 — ❌ Очередь перегружена + возврат"]
    RUN -.->|ошибка провайдера или S3| C8["C8 — ❌ Завершилось с ошибкой + возврат"]
    RUN --> C9["C9 — ✅ Распознавание завершено<br/>+ файл .txt + кнопки действий"]
    C9 --> C10["C10 — Оцените качество, кнопки 1–5 ⭐"]
    C10 -.->|оценка 1–2| D5["D5 — Расскажите, что пошло не так?"]
    D5 --> A2["A2 — Спасибо! Ваш отзыв поможет"]
    C9 -->|Отправить текстом| D1["D1 — текст в чат кусками"]
    C9 -->|Создать конспект| D2["D2 — ⏳ Создаю конспект → 📝 Конспект"]
    C9 -->|Знаки препинания| D3["D3 — ⏳ Оформляю текст → ✏️ Текст оформлен + файл"]
    C9 -->|С таймкодами| D4["D4 — выбор формата → файл .txt .srt .vtt"]
```

## Схема 3. Пополнение баланса

```mermaid
flowchart TD
    T([Команда /topup или клавиатура сумм из B7 C2]) --> E1["E1 — оферта + Выберите сумму<br/>кнопки: 50 / 100 / 250 / 500 ₽ / Отменить"]
    E1 -->|выбрал сумму| E2["E2 — Сумма пополнения: N ₽"]
    E1 -->|Отменить| E5["E5 — 🚫 Пополнение отменено"]
    E2 --> E3["E3 — Счёт на N ₽ создан<br/>кнопки: 💳 Оплатить / Отменить платёж"]
    E2 -.->|Тинькофф не ответил| E4["E4 — Не удалось создать форму оплаты"]
    E3 -->|оплата прошла| E6["E6 — ✅ Платёж успешно завершён"]
    E3 -.->|отказ банка| E7["E7 — 🚫 отклонён / ⌛ истекло время"]
    E3 -.->|прошло 3 часа| E8["E8 — Пополнение отменено автоматически"]
    E3 -->|Отменить платёж| E5
```

## Отличия Max

* Жирные заголовки (B6, C8, C9) шлются с `format=html` и откатываются на плоский текст, если Max отверг HTML. В TG жирное надёжно через `parse_mode=HTML`.
* Приветствие A1: подарочная строка без жирного; A3 отсутствует — сообщение без файла всегда отвечает приветствием.
* A7 (пересланное голосовое) есть только в Max.
* B9: свои формулировки — «❌ Не удалось получить файл от Max» / «❌ Не удалось загрузить файл».
* B6 в Max — новое сообщение (не редактирование), затем статус-сообщение удаляется.
