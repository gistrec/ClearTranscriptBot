# Эвристики качества распознавания

Свод эвристик, которыми бот улучшает результат WhisperX: адаптация входа, чистка вывода,
детекция брака с автовозвратом/повтором. Для каждой — что делает, как часто срабатывает,
доля ложных срабатываний и эффект. Цифры замерены по проду.

> По состоянию на **2026-06-27**. База: **12 202** записи `provider=replicate`, из них
> **11 217** с `result_json`, **11 147** completed, **35** rejected; `mean_volume_db`
> заполнен у **3 263** (фича новее таблицы). Все замеры — read-only по прод-MySQL.

**«False positives» здесь** — как часто эвристика бьёт по *хорошему* результату: срезает
настоящую речь, ошибочно возвращает деньги за нормальную расшифровку, флажит реальный
иностранный язык как мисдетект и т.п. Это цена эвристики; «Эффект» — её польза.

## A. Вход — адаптация перед распознаванием

### Замер громкости
`utils/ffmpeg.py:get_mean_volume` (ffmpeg `volumedetect`). Инфраструктура: меряет среднюю
громкость в dB, результат идёт в чувствительный VAD. Конвертация в OGG — `convert_to_ogg`;
нормализации громкости нет (предусиление было закрытым тупиком, см. память
`project_quiet_audio_research`).

- **Срабатывает:** на всех записях (замерено у 3 263).
- **False positives:** нет — это измерение, не решение.
- **Эффект:** даёт сигнал для чувствительного VAD.
- **Вывод:** работает, трогать нечего.

### Чувствительный VAD для тихих
`providers/replicate.py:start_transcription`, порог `QUIET_MEAN_VOLUME_DB = -35`. При
`mean_volume_db < -35` ставим `vad_onset=0.35 / vad_offset=0.25` вместо дефолтных
`0.5 / 0.363`, иначе дефолт байт-в-байт.

- **Срабатывает:** 195 / 3 263 (~6%) записей с громкостью.
- **False positives:** хорошая запись тише −35, которой чувствительный VAD навредил.
  Напрямую не наблюдаемо; по построению ≈0 — порог −35 при самой тихой *хорошей* записи
  −32.8 оставляет буфер ~2.2 dB (в этой зоне −35…−32.8 лежат 108 записей, и они остаются на
  дефолтном VAD).
- **Эффект:** тихие записи всё равно в среднем 2.94 (n=16) против 4.32 (n=245) у негромких —
  тихий звук остаётся сложным, но чувствительный VAD вытягивает речь, которую дефолтный VAD
  выбросил бы целиком (валидировано на жалобах в полосе −38…−91 dB).
- **Вывод:** работает, FP пренебрежимо мал — оставить как есть.

### Усиление детекции языка
`providers/replicate.py`, payload `language_detection_min_prob=0.9`,
`language_detection_max_tries=10`. Применяется всегда.

- **Срабатывает:** всегда (это конфиг, не условие).
- **False positives:** нет.
- **Эффект:** WhisperX увереннее определяет язык → меньше входных ошибок (держит
  `is_wrong_language` редким).
- **Вывод:** дёшево и полезно, оставить.

## B. Чистка вывода

Единая точка — `utils/timecodes.py:is_phantom_segment` (её же зовёт
`providers/replicate.py:_segments`, поэтому чистка применяется и к тексту, и к таймкодам, и к
детектору галлюцинаций).

### `_PHANTOM_CREDIT_RE`
Титры-галлюцинации `DimaTorzok | Редактор субтитров | Корректор А.Егорова`, подстрочный матч.

- **Срабатывает:** 493 / 11 217 (4.4%) — совпадает с прежней оценкой «~4%».
- **False positives:** ≈0. Эти точные имена не встречаются в живой речи, поэтому подстрочный
  матч безопасен.
- **Эффект:** убирает кредит-строку примерно из каждой 23-й расшифровки.
- **Вывод:** безопасно и полезно, оставить.

### Пустые сегменты
`extract_segments` дропает сегменты без текста.

- **Срабатывает:** по мере появления пустых сегментов.
- **False positives:** 0 (нечего терять в пустом сегменте).
- **Эффект:** не мусорит пустыми строками.
- **Вывод:** тривиально, оставить.

### `_PHANTOM_CONTINUATION_RE`
Внедрено в этой итерации (коммит `ceee5bf`). Анкорный матч «Продолжение следует» как
**целого** сегмента.

- **Срабатывает:** 468 / 471 вхождений фразы (standalone).
- **False positives:** ~1–5 из 471 — хвостовая отбивка на ТВ/видео, семантический пустяк.
- **Эффект:** убирает галлюцинацию «Продолжение следует» из вывода; заодно перестаёт тянуть
  вниз средний logprob (см. `looks_like_hallucination`).
- **Вывод:** внедрено, работает.

## C. Детекция брака → автодействие

`schedulers/transcription.py` после получения результата проверяет два сигнала.

### `looks_like_hallucination`
`providers/replicate.py:looks_like_hallucination`. Срабатывает если **≥50% сегментов** имеют
`avg_logprob < -0.5` ИЛИ длинный (≥15 симв.) сегмент повторён ≥3 раза и составляет ≥50%
сегментов. Срабатывание → `STATUS_REJECTED` + автовозврат денег + честное сообщение. (Пустой
результат отклоняется отдельной веткой «не нашлось разборчивой речи», не этой эвристикой.)

- **Срабатывает:** 35 / ~11 220 (~0.3%).
- **Что чинили (этот коммит):** прежняя версия флажила по **среднему** `avg_logprob < -0.5`, а
  среднее утягивает вниз меньшинство плохих сегментов — и возвращала деньги за длинные
  **связные** записи: 20158 (лекция 2.7 ч, 363 сегмента, лишь 4 с logprob ≤ −1), 20129 (звонок
  про страховку), 20109 (часовой звонок). Калибровка по проду: у мусора **45–100%** сегментов
  ниже −0.5, у связных/хороших — **0–8%**. Перешли на **долю** низко-logprob сегментов (≥50%)
  — чистое разделение, длинные связные больше не отклоняются.
- **False positives после фикса:** низко. «≥50% сегментов невнятные» не задевает связные
  (≤8%) и хорошие (0–3%). Компромисс: умеренный мусор с долей <50% (напр. 20970, 45%) теперь
  проходит — сознательный приоритет «не возвращать деньги за хорошее».
- **Эффект:** ловит сплошной брак и петли, перестав штрафовать длинную связную речь.
- **Вывод: доработано.** Петля и ветка пустого результата без изменений.

### `is_wrong_language`
`providers/replicate.py:is_wrong_language`. Срабатывает если `detected_language` ∈
{uk, nn, kk, ko, zh, es} ИЛИ >20% символов в чужом письме (CJK/арабица/хангыль…).
Проверяется **раньше** `looks_like_hallucination` → предлагает бесплатный пикер «перераспознать
на другом языке» вместо возврата.

- **Срабатывает:** 178 / 11 217 (1.6%) по коду языка (uk 56, nn 43, zh 26, es 21, ko 19,
  kk 13) + ветка по чужому письму (отдельно не считалась).
- **False positives:** низко. Флагнутый набор в среднем имеет **рейтинг 2.33**, тогда как
  настоящие иностранные загрузки (de/fr/ar/tr/it/ja) — **4.59**. То есть wrong-набор — это
  почти всегда мисдетект русского, а не реальный чужой язык; ложный флаг = редкая настоящая
  загрузка на uk/kk/…, которая и так распозналась бы хорошо.
- **Эффект:** превращает результат на «не том языке» в один тап повторного распознавания.
- **Вывод:** работает, FP низкий — оставить.

## Кандидаты / отложено

### «Спасибо за просмотр» — **не внедрять**
Разбор по аналогии с «Продолжение следует»:

- Всего **28 / 11 217 (0.25%)** вхождений — в ~16 раз меньше объёма.
- Доля настоящих **высокая**: ~6–10 из 28 — реальные аутро видео: встроенные
  «…спасибо за просмотр, встретимся в следующем ролике» (id 14988/6931/949),
  «…схема проезда на экране. Спасибо за просмотр.» (13284), полноценные длинные аутро
  (16765 с logprob −0.07, 15149).
- Голые standalone-вхождения в основном галлюцинации (длительности 5–30 с, низкий logprob,
  кластеры с DimaTorzok), но строгий анкор всё равно мог бы срезать настоящее хвостовое
  «Спасибо за просмотр!».
- **Вывод:** риск FP заметно выше, чем у «Продолжение следует» (там 1/471), при многократно
  меньшем объёме. Не внедрять. Если когда-нибудь добавлять — только строгий анкор на **весь**
  сегмент **плюс** вторичный сигнал (нереальная длительность >4 с или низкий logprob).

### Прочие отложенные рычаги (ресёрч есть в памяти)
- **Маркеры дыр** — показывать `[02:35 – 05:43] фрагмент без распознанной речи` вместо немого
  скачка таймкода на участках, выброшенных VAD.
- **Фолбэк incredibly-fast-whisper** — узкий, с принудительным русским, на сигнатуру сбоя;
  `project_incredibly_fast_whisper_research` (основную модель не менять).
- **Диаризация** — кто говорит; `project_diarization_research`.
